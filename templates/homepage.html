Configuration

Installations analyzed by Feedback Pipeline are defined in a git repository in YAML format.

Installations

Installations define what is actually analyzed â€” a set of packages representing an app or an image.

      
document: feedback-pipeline-install
version: 0
data:
    name: some-id
    summary: "Visible name"
    base: empty
    pkgs:
      - pkg1
      - pkg2
      - pkg3

Views

Views group together multiple installations. Defines a base installation (such as a base image) and one or more additional installations installed on top of the base.

        
document: feedback-pipeline-view
version: 0
data:
    name: some-id
    title: "Visible name"
    summary: "visible summary"
    releases:
        - 30
        - 31
    base_installation: installation-id
    installations:
        - another-id
        - yet-another-id
